<ToolDescription>

<ToolName>
Memory
</ToolName>

<ToolUse>
The Memory tool lets you store persistent memories that can be retrieved later
To use this tool, you have 5 modes, the first 2 are used to interact with timestamped memories : 
- record : put the memory you want to record within the <in_data></in_data> tags, like this :

<call>
<tool>
Memory
</tool>
<action>
record
</action>
<in_data>
I learned that the user's phone broke today
The user told me that the weather was bad today
</in_data>
</call>

the "record" mode will automatically add a timestamp to memories for future retrieval

- retrieve : retrieve memories recorded between certain timestamps with different formats in the first line of the <in_data></in_data> section, 2 formats are available :
    - last X {days/weeks/months/years} : this will retrieve memories recorded in the specified time to the current chat time, this is good for quickly gathering large amounts of memories to look back on for general information
    - from:YYYY-MM-DD to:YYYY-MM-DD : this will retrieve memories recorded between the 2 specified dates (inclusively), this is best for finding specific memories when you have more details on their likely dates
        - note : for that second format, you can also use "start" for the "from" to start from the date of the earliest memory recorded if you need to search through very old memories

a call of the "retrieve" mode may look like this with the first format :

<call>
<tool>
Memory
</tool>
<action>
retrieve
</action>
<in_data>
last 3 weeks
</in_data>
</call>

and with the second format, the <in_data></in_data> section may look like this instead (dates chosen randomly for demonstration purposes) :
<in_data>
from:2024-05-08 to:2025-03-17
</in_data>

or even :
<in_data>
from:start to:2025-09-28
</in_data>

There are 3 more modes, used to interact with "persistent" memory, which is memory that will always be easily found in a single command. It should be prioritised to retain information that is important to always have in mind, such as names or interests for example. Those 3 persistent modes are as follows:

- persistent_get : this returns either all of your persistent memory, or up to the N first lines as specified in the <in_data></in_data> section which may be empty if you need to get the whole persistent memory, for example :

<call>
<tool>
Memory
</tool>
<action>
persistent_get
</action>
<in_data>
</in_data>
</call>

- persistent_add : this lets you add lines to persistent memory, they will always be added at the end of the current memory. all lines in the <in_data></in_data> section will be added, here's an example :

<call>
<tool>
Memory
</tool>
<action>
persistent_add
</action>
<in_data>
The user's name is {user's name}
Their favorite color is red
</in_data>
</call>

- persistent_remove : this lets you remove individual lines from persistent memory by specifying which ones to remove on the first line of the <in_data></in_data> section, like so : 
<call>
<tool>
Memory
</tool>
<action>
persistent_remove
</action>
<in_data>
5 12 17 18 10
</in_data>
</call>

</ToolUse>

<ToolOutput>
This tool outputs the date of recording in the "record" mode in the YYYY/MM/DD format, like this :

<output_data>
Memory successfully recorded on 2026-02-20
</output_data>

for the "retrieve" mode in the second example, all retrieved memories will be provided with their timestamp, like this :

<output_data>
Date: 2024-11-21
{contents of this memory}
----
Date: 2025-01-15
{contents of this memory}
----
</output_data>

if no data is found within the provided search period, you will be notified

for the "persistent_get" mode, the persistent memory will be returned with each line numbered automatically, like this (using the memory from the persistent_add example) : 

<output_data>
0 The user's name is {user's name}
1 Their favorite color is red
</output_data>

for the "persistent_add" mode, the output will return the line numbers of the newly added lines, like this for the example above :

<output_data>
Lines successfully added with numbers 0, 1
</output_data>

for the "persistent_remove" mode, the output will specify if the operation was a success

<output_data>
Lines successfully removed
</output_data>

</ToolOutput>

<ToolErrors>
There will be 2 types of errors :
- Unreachable Database : this will happen when the database can't be reached for either mode's function
- Wrong Format : this will happen when the provided search terms for the "retrieve" mode are wrongly formatted
</ToolErrors>

</ToolDescription>
